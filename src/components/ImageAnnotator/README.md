**COCO**

全称 Common Objects in Context，微软做的大型图像数据集，顺带定义了一套标注格式，现在已经成为行业标准。

```js
{
  "images": [
    { "id": 1, "file_name": "cat.jpg", "width": 640, "height": 480 }],
  "categories": [
    { "id": 1, "name": "cat" },
    { "id": 2, "name": "dog" }
  ],
  "annotations": [
    {
        "id": 1,
        "image_id": 1,
        "category_id": 1,
        "bbox": [100, 50, 200, 150],
        "segmentation": [[100, 50, 300, 50, 300, 200, 100, 200]]
    }]
}
```

- bbox 是 [x, y, width, height]
- segmentation 是多边形顶点坐标
- 适合复杂场景，支持矩形框 ****+ 多边形分割

YOLO

全称 You Only Look Once，一个目标检测模型的名字，它定义了自己的标注格式，非常简洁。

每张图对应一个 .txt 文件，每行一个目标：

```js
# 格式：类别id  中心x  中心y  宽  高（全部是0~1的相对坐标）
0 0.5 0.4 0.3 0.25
1 0.2 0.7 0.1 0.15
```

比如图片 640×480，一个猫的框：

```js
# 类别0，中心在(320,192)，框大小192×120
0 0.5 0.4 0.3 0.25
```

- 坐标全是相对值（除以图片宽高），所以换图片尺寸不用重新算
- 格式极简，适合直接喂给 YOLO 模型训练

表格对比：

|          | COCO                    | YOLO             |
| -------- | ----------------------- | ---------------- |
| 格式     | JSON                    | TXT              |
| 坐标     | 绝对像素值              | 相对比例 0~1     |
| 支持分割 | 多边形                  | 只有矩形框       |
| 适合场景 | 复杂标注、多种任务      | 快速训练检测模型 |
| 模型生态 | Detectron2、MMDetection | YOLOv5/v8/v11    |

实际项目里，标注完通常会同时导出两种格式，COCO 用来存档，YOLO 用来直接训练模型。
